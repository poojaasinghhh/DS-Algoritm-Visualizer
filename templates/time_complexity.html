Understanding Time Complexity

Introduction to Time Complexity

Time complexity is a computational concept that helps us understand the amount of time an algorithm takes to run as a function of the length of the input. It allows us to analyze and compare the efficiency of different algorithms.

Big O Notation

Big O notation is used to describe the upper bound of the time complexity. It gives the worst-case scenario of how an algorithm performs as the input size grows. For example: 

function example(arr) {
    for (let i = 0; i < arr.length; i++) {
        console.log(arr[i]);
    }
}
// Time Complexity: O(n)

Big Ω Notation

Big Omega (Ω) notation is used to describe the lower bound of the time complexity. It gives the best-case scenario. For example: 

function example(arr) {
    return Math.max(...arr);
}
// Time Complexity: Ω(1)

Big Θ Notation

Big Theta (Θ) notation provides a tight bound on the time complexity. It gives both the best and worst-case scenarios. For example: 

function sum(arr) {
    let total = 0;
    for (let i = 0; i < arr.length; i++) {
        total += arr[i];
    }
    return total;
}
// Time Complexity: Θ(n)
